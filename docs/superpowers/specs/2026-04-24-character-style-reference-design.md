# 项目级视觉参考图体系 — 产品与工程设计

> 文档版本:v2.0 · 2026-04-24
> 范围:角色设定页、场景设定页、角色/场景资产生成链路
> 关联模块:`CharacterAssetsPanel`、`SceneAssetsPanel`、`PromptProfileCard`、项目聚合接口、角色/场景出图任务、OBS 资产存储
> 设计原则:统一视觉参考图是项目级风格母版,不绑定具体角色或具体场景;所有远程 AI 调用必须异步;生成图必须落入项目资产域并可刷新恢复。

---

## 1. 背景

当前项目已经有“角色统一视觉设定”和“场景统一视觉设定”两类文本设定,分别用于约束人物画风与场景风格。但只靠文本约束仍存在明显不稳定:

- 角色参考图之间可能出现脸型比例、身体比例、服装质感和渲染风格漂移。
- 单张角色图同时承担全身形象和面部细节,后续镜头生成时不够稳定。
- 场景参考图可能混入人物,导致后续镜头把无关人物当作场景资产的一部分。
- 用户缺少可视化的项目级风格锚点,很难判断统一设定是否已经锁住。

本设计新增两个项目级视觉母版:

- `统一角色形象参考图`:白底、正面、全身、单人示范图,用于固定全项目人物画风、比例、服装质感。
- `统一场景视觉参考图`:无人物、纯场景示范图,用于固定全项目空间设计、色彩光影、建筑/道具质感。

同时调整具体资产生成:

- 每个角色生成两张白底图:一张全身图,一张大头像图。
- 每个场景生成图中明确禁止出现人物。

## 2. 目标与非目标

### 2.1 目标

- 在角色设定页“角色统一视觉设定”旁边新增项目级角色风格参考图。
- 在场景设定页“场景统一视觉设定”旁边新增项目级场景视觉参考图。
- 角色生成输出两张图:
  - 白底正面全身图,用于全身造型、服装、体态、比例。
  - 白底大头像图,用于脸型、五官、发型、妆造细节。
- 场景生成图必须禁止出现人物,只输出环境、建筑、空间、道具和氛围。
- 所有生成结果保存到项目/资产数据中,刷新页面后可恢复。
- 后续镜头生成可使用角色全身图、角色头像图、场景无人图作为参考候选。
- 所有远程 AI / 出图 / 上传动作走异步 job,HTTP 请求只创建任务。

### 2.2 非目标

- 不做多版本风格母版管理;MVP 只保存当前生效图。
- 不做跨项目风格母版复用。
- 不做用户本地图片上传作为视觉母版;如后续支持上传,必须先通过资产存储和内容安全校验。
- 不把项目级角色风格母版注册到火山人像库;它不是具体人物身份。
- 不改变项目阶段流转,只复用现有角色/场景阶段权限。
- 不把场景图的人物遮罩、自动擦除作为本期兜底能力;本期通过 prompt 和验收阻止人物出现。

## 3. 产品体验

### 3.1 角色设定页

角色设定页顶部改为双栏:

```text
角色设定
├── 左侧:角色统一视觉设定
│   ├── AI 生成建议
│   ├── 设定文本编辑
│   ├── 保存草稿
│   └── 确认并生成角色
└── 右侧:统一角色形象参考图
    ├── 白底正面全身预览
    ├── 状态:未生成 / 生成中 / 已生成 / 失败
    └── 操作:生成参考图 / 重新生成 / 重试
```

卡片说明:

```text
用于统一后续角色参考图的人物比例、画风、服装质感和白底全身形象,不绑定具体剧情角色。
```

### 3.2 场景设定页

场景设定页顶部同样改为双栏:

```text
场景设定
├── 左侧:场景统一视觉设定
│   ├── AI 生成建议
│   ├── 设定文本编辑
│   ├── 保存草稿
│   └── 确认并生成场景
└── 右侧:统一场景视觉参考图
    ├── 无人物场景预览
    ├── 状态:未生成 / 生成中 / 已生成 / 失败
    └── 操作:生成参考图 / 重新生成 / 重试
```

卡片说明:

```text
用于统一后续场景参考图的空间结构、色彩光影、建筑材质和时代氛围;画面中不允许出现人物。
```

### 3.3 具体角色资产展示

角色详情页从单张参考图变为双图展示:

```text
角色详情
├── 全身参考图
│   └── 白底、正面、全身、服装体态完整
└── 头像参考图
    └── 白底、正面或轻微三分之二角度、大头像、五官发型清晰
```

角色列表仍按角色维度展示,不拆成两个资产项。进入角色详情后展示双图。

### 3.4 具体场景资产展示

场景详情页继续展示一张场景参考图,但图像要求改为:

- 不出现人物。
- 不出现人群剪影。
- 不出现主体角色背影。
- 允许出现建筑、家具、器物、灯火、自然景观、天气和空间光影。

## 4. 状态与交互

### 4.1 统一视觉参考图状态

| 状态 | UI 表现 | 可用操作 |
| --- | --- | --- |
| 未生成 | 占位图 + 简短说明 | 生成参考图 |
| 生成中 | 进度条 + “正在生成...” | 禁用重复提交 |
| 已生成 | 展示图片 + 最近更新时间 | 重新生成 |
| 失败 | 错误文案 + 失败原因 | 重试 |
| 阶段锁定 | 按钮禁用 | 提示回退阶段 |

### 4.2 重生成规则

- 重生成成功前,继续展示上一张成功图。
- 重生成失败不覆盖上一张成功图。
- 成功后替换当前生效图,并保存新 prompt snapshot。

### 4.3 阶段权限

- 角色风格母版:与 `canEditCharacters` 一致,只允许在 `storyboard_ready` 阶段编辑/生成。
- 场景风格母版:与 `canEditScenes` 一致,只允许在 `characters_locked` 阶段编辑/生成。
- 具体角色双图生成:沿用现有角色生成阶段权限。
- 具体场景无人图生成:沿用现有场景生成阶段权限。
- 如果项目已经推进到更晚阶段,重做风格母版必须先通过阶段回退回到对应可编辑阶段;本期不提供跨阶段强制重生成入口。

## 5. 数据设计

### 5.0 资产边界

本期涉及四类图片资产,职责不同:

| 资产类型 | 存储位置 | 用途 | 是否本期新增 |
| --- | --- | --- | --- |
| 项目级角色风格母版 | `projects.character_style_reference_*` | 统一全项目人物画风、比例、白底全身质感 | 是 |
| 项目级场景视觉母版 | `projects.scene_style_reference_*` | 统一全项目空间、光影、材质,且不含人物 | 是 |
| 手动参考资产 | `project_reference_assets` | 用户手动加入镜头参考候选图 | 已存在,本期不复用为风格母版 |
| 具体角色/场景资产 | `characters` / `scenes` | 绑定具体角色或具体场景的参考图 | 扩展角色双图 |

本期风格母版不使用现有 `project_reference_assets` 表;该表继续只服务镜头生成中的手动参考候选。

### 5.1 Project 新增字段

MVP 将项目级视觉母版保存到 `projects` 表。

```text
character_style_reference_image_url: string | null
character_style_reference_prompt: JSON | null
character_style_reference_status: string | null
character_style_reference_error: string | null

scene_style_reference_image_url: string | null
scene_style_reference_prompt: JSON | null
scene_style_reference_status: string | null
scene_style_reference_error: string | null
```

字段含义:

- `*_image_url`: OBS object key,聚合接口输出时转为公网可访问 URL。
- `*_prompt`: 生成时的 prompt snapshot,用于排查和复现。
- `*_status`: `empty | running | succeeded | failed`。
- `*_error`: 最近一次失败原因。

### 5.2 Character 新增字段

角色从单图升级为双图。为兼容旧数据,保留原 `reference_image_url` 作为全身图别名或兼容字段。

```text
reference_image_url: string | null
full_body_image_url: string | null
headshot_image_url: string | null
```

兼容规则:

- 新数据写入 `full_body_image_url` 和 `headshot_image_url`。
- MVP 阶段继续同步写入 `reference_image_url=full_body_image_url`,保证旧前端、旧接口和旧镜头引用不立即崩;后续若废弃旧字段,需要单独做字段迁移和调用方清理。
- 聚合接口输出 `reference_image_url` 时优先取 `full_body_image_url`,再 fallback 到旧字段。

### 5.3 Scene 字段

场景仍保留:

```text
reference_image_url: string | null
```

但生成和重生成必须保证该图为“无人场景图”。

### 5.4 聚合响应

`ProjectData` 增加:

```ts
characterStyleReference: {
  imageUrl: string | null;
  prompt: string | null;
  status: "empty" | "running" | "succeeded" | "failed";
  error: string | null;
}

sceneStyleReference: {
  imageUrl: string | null;
  prompt: string | null;
  status: "empty" | "running" | "succeeded" | "failed";
  error: string | null;
}
```

`CharacterAsset` 增加:

```ts
full_body_image_url: string | null;
headshot_image_url: string | null;
```

旧项目没有这些字段时,前端按空值处理。

## 6. API 设计

### 6.1 生成角色风格母版

```http
POST /api/v1/projects/{project_id}/character-style-reference/generate
```

返回:

```json
{ "job_id": "01..." }
```

Job kind:

```text
gen_character_style_reference
```

### 6.2 生成场景风格母版

```http
POST /api/v1/projects/{project_id}/scene-style-reference/generate
```

返回:

```json
{ "job_id": "01..." }
```

Job kind:

```text
gen_scene_style_reference
```

### 6.3 查询

不新增单独查询接口,通过现有项目聚合接口返回:

- `characterStyleReference`
- `sceneStyleReference`
- 角色双图字段
- 场景无人图字段

### 6.4 Job 进度建议

统一视觉母版:

- 10: 创建任务
- 30: 组装 prompt
- 70: 调用出图模型
- 90: 上传 OBS / 写回项目
- 100: 完成

角色双图生成:

- 10: 抽取/读取角色
- 30: 生成全身图
- 60: 生成头像图
- 90: 上传并写回角色
- 100: 完成

场景无人图生成:

- 10: 读取场景
- 35: 组装无人场景 prompt
- 75: 调用出图模型
- 90: 上传并写回场景
- 100: 完成

图片尺寸:

- 项目级角色风格母版、角色全身图、角色头像图:使用竖版尺寸,推荐 `768x1344` 或同等 9:16 比例。
- 项目级场景视觉母版、具体场景无人图:使用横版尺寸,推荐 `1344x768` 或同等 16:9 比例。
- prompt 中的构图比例必须与 API `size` 参数一致,不得使用 `1024x1024` 生成本期这些资产。

## 7. Prompt 设计

本节中的 `{{...}}` 为说明占位符。实现时使用 Python 字符串组装,不引入模板引擎;占位内容必须来自当前项目、角色或场景的已落库字段。

### 7.1 角色风格母版 Prompt

```text
尺寸:768x1344 或同等 9:16 竖版比例。
9:16竖版构图。生成项目级角色风格参考图,作为后续所有角色参考图的视觉母版,不是具体剧情角色。画面为单人正面全身角色设定图,人物居中站立,头顶到脚底完整可见,双臂自然下垂或轻微收拢,姿态端正自然,不做夸张动作。背景为纯白色,无场景、无道具、无文字、无边框。

人物要求:根据项目级角色统一视觉设定生成一个中性示范人物,不出现明确姓名、身份标识或剧情符号。面部需要体现统一的脸型比例、五官风格、眼神气质、发型质感和肤色处理;服装需要体现统一的时代风格、层次结构、布料材质、纹样密度、新旧状态和整体色彩体系。身体比例稳定,肩颈、手部、腿部和脚部结构清楚,服饰轮廓完整。

项目级角色统一视觉设定:
{{character_prompt_profile}}

光影要求:白底棚拍式柔和主光,正面与侧前方补光均衡,保留轻微轮廓光,面部和服装细节清晰,不要强烈戏剧化阴影。画面质感为高质量角色设定图,线条、材质、皮肤、布料和饰件细节干净稳定。

禁止:多人、复杂背景、半身像、头像裁切、侧脸、背影、过度动态姿势、身体遮挡、手脚缺失、文字、字幕、LOGO、水印、背景非白色、道具抢画面、风格漂移。
```

### 7.2 场景风格母版 Prompt

```text
尺寸:1344x768 或同等 16:9 横版比例。
16:9横版构图。生成项目级场景视觉参考图,作为后续所有场景参考图的空间风格母版,不是具体剧情镜头。画面只呈现场景环境,绝对不出现任何人物。重点表现空间结构、时代建筑、环境道具、材质纹理、灯光氛围、天气状态、色彩体系和镜头景深。

场景要求:根据项目级场景统一视觉设定生成一个示范场景,不绑定具体剧情事件,不出现具体角色。构图使用宽阔清晰的场景视角,前景、中景、远景层次明确,建筑轮廓稳定,空间入口、墙面、地面、家具或环境道具关系清楚。画面需要体现统一的时代气质、建筑材质、装饰语言、色彩冷暖和光影方向,适合作为后续场景资产的风格参考。

项目级场景统一视觉设定:
{{scene_prompt_profile}}

光影要求:主光源方向明确,环境补光自然,材质表面的反光、潮湿、磨损、灰尘或年代痕迹清楚。色调与项目设定一致,不要过度饱和,不要出现现代错置元素。画面质感为高质量场景概念设计图,结构稳定,细节丰富。

禁止:人物、人脸、人群、背影、剪影、手脚身体局部、角色站位、人物衣物残影、文字、字幕、LOGO、水印、现代错置元素、结构混乱、透视崩坏、风格漂移。
```

### 7.3 具体角色全身图 Prompt

```text
尺寸:768x1344 或同等 9:16 竖版比例。
9:16竖版构图。生成角色全身参考图,用于后续分镜和视频生成中的人物一致性锁定。画面为单人白底正面全身角色设定图,角色居中站立,头顶到脚底完整可见,姿态自然稳定,双臂自然下垂或轻微收拢,不做夸张动作。

角色名称:
{{character.name}}

角色简介:
{{character.summary}}

角色详述:
{{character.description}}

项目级角色风格参考图使用规则:
参考项目级角色风格母版的人物画风、身体比例、服装材质、白底全身构图和渲染质感,但不要继承母版示范人物的身份、姓名、脸部特征或剧情关系。当前角色的五官、年龄感、身份气质、服装款式、服饰新旧状态和随身标识必须严格来自角色简介与角色详述。

面部要求:脸型、眼型、眉形、鼻型、唇形、肤色、眼神和表情需要清晰稳定,符合角色身份与年龄。发型、头饰和配饰必须与角色描述一致,不能凭空增加高辨识道具。

服装要求:从头到脚完整描述并呈现上装、下装、鞋履、腰带、披风、甲胄、袍服或其它服饰结构;布料材质、金属质感、纹样、磨损、雨痕、血污、灰尘等状态忠实体现角色设定,不要美化或降级。

光影要求:白底棚拍式柔和主光,面部、手部、衣褶、材质和轮廓清晰,角色边缘有轻微轮廓光。画面质感为高质量角色设定图,结构稳定,细节干净。

禁止:多人、复杂背景、半身像、头像裁切、侧脸、背影、身体遮挡、手脚缺失、额外道具抢画面、文字、字幕、LOGO、水印、服装风格漂移、脸部与角色设定不符。
```

### 7.4 具体角色头像图 Prompt

```text
尺寸:768x1344 或同等 9:16 竖版比例。
9:16竖版构图。生成角色大头像参考图,用于锁定后续镜头中的脸型、五官、发型、眼神和妆造细节。画面为单人白底大头像或胸像近景,角色面部占画面主体,头发完整可见,肩颈和服饰领口适度入画。

角色名称:
{{character.name}}

角色简介:
{{character.summary}}

角色详述:
{{character.description}}

项目级角色风格参考图使用规则:
参考项目级角色风格母版的人物画风、脸型比例、五官风格、发型质感、服饰领口材质和渲染质感,但不要继承母版示范人物的身份、姓名、脸部特征或剧情关系。当前角色的年龄感、身份气质、面部辨识度和情绪底色必须严格来自角色简介与角色详述。

面部要求:脸型轮廓明确,眼型、瞳色或眼神方向稳定,眉形、鼻梁、唇形、下颌线、肤色和皮肤质感清楚。表情保持角色设定中的基础神态,不要夸张大笑或强烈扭曲。发型、发冠、发簪、帽饰、耳饰、胡须或伤痕等高辨识元素必须清楚可见。

服饰要求:领口、肩部和上半身局部服饰需要体现角色身份和材质,与全身图服装体系一致。背景为纯白色,无场景、无道具、无文字。

光影要求:柔和正面主光配合轻微侧前方补光,眼部高光自然,面部阴影不过重,五官边界清晰。画面质感为高质量角色头像设定图,面部稳定,细节丰富。
```

## 9. 错误处理

- 没有统一视觉设定时仍可生成母版,但 prompt 使用项目类型、摘要和默认风格约束。
- 母版生成失败:写入对应 `*_status=failed` 和错误信息,前端展示重试。
- OBS 上传失败:任务失败,不覆盖上一张成功母版。
- 角色双图部分失败:
  - 全身图成功、头像图失败时,该角色生成任务标记 failed,但保留已成功上传图并展示重试。
  - 实现上每张图上传成功后立即写库并提交;重试时优先补缺失图,如果两张都已存在且用户点击重新生成,则全量重生成。
- 场景图如果生成出人物,本期不做自动识别删除;用户通过“重新生成”处理。后续可接入视觉审核。
- 阶段不允许时:返回 40301,前端给出回退阶段提示。

## 10. 验收标准

### 10.1 角色母版

- 角色设定顶部能看到“角色统一视觉设定”和“统一角色形象参考图”并排展示。
- 点击生成后返回 job,页面展示进度,刷新后能重新接管 running job。
- 生成成功后展示白底、正面、全身、单人参考图。
- 重生成失败不会清掉上一张成功图。

### 10.2 场景母版

- 场景设定顶部能看到“场景统一视觉设定”和“统一场景视觉参考图”并排展示。
- 生成成功后展示无人物场景参考图。
- 场景母版 prompt 明确包含“禁止出现人物、人脸、人群、背影、剪影、身体局部”。

### 10.3 角色双图

- 每个角色详情展示两张图:全身参考图和头像参考图。
- 两张图均为白底。
- 全身图完整展示头顶到脚底。
- 头像图清晰展示脸型、五官、发型和领口服饰细节。
- 后续镜头参考候选中能区分“角色全身图”和“角色头像图”。

### 10.4 场景无人图

- 每个场景只输出环境图,不得出现人物。
- 场景 prompt 和重生成 prompt 都包含禁止人物约束。
- 后续镜头参考候选中的场景图不包含人物。

### 10.5 工程约束

- 所有远程 AI 调用均在 Celery job 中执行,HTTP 请求不阻塞等待出图。
- 项目刷新后母版图、角色双图、场景无人图均能恢复展示。
- 老项目缺少新增字段时页面不报错,按空态展示。
