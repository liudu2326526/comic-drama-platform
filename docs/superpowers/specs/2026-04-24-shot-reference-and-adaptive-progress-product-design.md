# 镜头生成参考图绑定与自适应进度条 — 产品设计文档

> 文档版本:v1.0 · 2026-04-24  
> 范围:M3b+ 镜头生成体验增强  
> 关联模块:`GenerationPanel`、镜头草稿、参考图候选、视频生成 job 进度  
> 设计原则:不改变现有项目阶段流转;不把远程 AI 调用放入 HTTP 同步请求;参考图与提示词绑定关系必须可保存、可复现、可追溯。

---

## 1. 背景

当前 M3b 的镜头生成链路已经支持“生成草稿 → 编辑提示词/参考图 → 确认生成视频 → 查看历史版本”。但参考图仍偏静态:系统自动给出候选后,用户只能删除,无法主动添加;提示词中的角色、场景描述也没有和具体参考图建立明确绑定。

此外,镜头视频生成的耗时受模型类型、分辨率、时长、排队情况影响较大。现有进度条主要展示任务内部写入的 `jobs.progress`,在长时间无回调或 provider 侧轮询阶段容易停滞,用户无法判断剩余等待时间。

本设计补齐两个产品能力:

- 镜头生成中,参考图可追加、可排序、可用 `@` 在提示词中引用。
- 进度条根据最近 N 条成功任务耗时做自适应估算,默认 N=20,可由配置调整。

---

## 2. 目标与非目标

### 2.1 目标

- 用户可以在镜头生成页为当前镜头追加参考图,来源包括角色、场景、镜头历史图和手动登记的项目资产图。
- 已选参考图以缩略图横排显示在镜头提示词上方,可删除、排序、点击插入引用。
- 用户在提示词输入 `@` 后弹出下拉选项,选择当前已选参考图并插入可读引用。
- 提交生成时,提示词、参考图列表、引用关系一起保存到版本快照,保证历史版本可复现。
- 视频生成进度展示更接近真实等待体验,基于最近 N 条同类成功任务耗时估算展示进度和剩余时间。
- 后端仍以 `update_job_progress` 作为唯一真实 job 状态写入口;自适应进度只作为展示增强。

### 2.2 非目标

- 不做完整富文本编辑器,本期仍使用 textarea。
- 不做多人协同编辑、评论、批注。
- 不做跨项目参考图库复用。
- 不改变项目阶段状态机,不新增镜头生成阶段。
- 不实现并行多镜头调度器;批量渲染仍属于 M3c/M4 范围。
- 不让手动参考图直接绕过内容安全和资产存储规则;本期只登记已经落在项目资产域内的图片 URL / OBS 对象,不接受任意外链作为可复用参考图。

---

## 3. 用户角色与使用场景

### 3.1 主要用户

短剧创作者、编导、AI 视频生成操作员。他们通常需要在每个镜头中明确告诉模型“这个人物参考哪张图”“这个场景参考哪张图”“这张手动图只是氛围参考”。

### 3.2 核心场景

1. 用户进入镜头生成页,选中一个镜头。
2. 系统已有自动草稿和默认参考图。
3. 用户发现还需要一张角色侧脸图或上一镜头尾帧作为连续性参考。
4. 用户点击“添加参考图”,从候选图库追加图片;如需手动图,先选择已经入库的项目资产图。
5. 参考图缩略图出现在提示词上方。
6. 用户在提示词中输入 `@`,选择“秦昭 正面参考图”。
7. 提示词中插入 `@秦昭`,提交生成时后端保存其与图片的绑定。
8. 生成过程中进度条根据历史耗时平滑推进,并显示预计剩余时间。

## 4. 信息架构

### 4.1 镜头生成面板结构

镜头生成主区域调整为:

```text
镜头生成
├── 左侧:镜头列表
└── 右侧:当前镜头工作区
    ├── 操作区:生成草稿 / 查看历史 / 生成视频
    ├── 参数区:时长 / 分辨率 / 模型类型
    ├── 进度区:当前生成进度、预计剩余时间
    ├── 参考图区:横向缩略图列表 + 添加按钮
    ├── 提示词区:textarea + @ 下拉
    └── 预览区:当前成功版本或生成中状态
```

### 4.2 参考图来源

| 来源 | 说明 | 是否默认入选 | 是否可追加 |
| --- | --- | --- | --- |
| 角色参考图 | M3a 角色资产图 | 按镜头文案自动选 | 是 |
| 场景参考图 | M3a 场景资产图 | 绑定场景优先 | 是 |
| 镜头历史图 | 当前项目内已生成镜头的首帧/尾帧/成功版本图 | 否 | 是 |
| 手动项目资产图 | 用户已放入当前项目资产域的参考图 | 否 | 是 |

手动项目资产图属于项目级资产,但可被多个镜头重复选择。本期不做跨项目资产复用。MVP 不接收任意公网图片 URL 作为长期参考图;如果后续补 multipart 上传,上传接口必须先完成资产存储与内容安全校验,再返回可登记的项目资产 URL。

## 5. 交互设计

### 5.1 参考图横排缩略图

参考图区域位于提示词输入框上方。

```text
参考图 4/6                                      + 添加
[长安殿 场景] [秦昭 角色] [暗巷 场景] [尾帧 历史]

镜头提示词
[ textarea ... ]
```

每张参考图卡片包含:

- 缩略图
- 名称
- 类型标识:角色 / 场景 / 历史 / 手动
- 删除按钮
- hover 时显示来源说明

支持操作:

- 点击缩略图:在当前光标位置插入对应 `@引用`。
- 排序控件:通过拖拽或上/下移按钮调整传给模型的图片顺序。
- 删除:从当前镜头已选参考图中移除。
- 查看大图:点击预览图或放大图标打开轻量预览。

默认上限建议为 6 张,由配置控制。达到上限后,“添加”按钮 disabled 并提示“当前模型最多支持 6 张参考图”。

### 5.2 添加参考图

点击“添加”打开参考图选择弹层。

弹层结构:

```text
添加参考图
├── 搜索框
├── 类型筛选:全部 / 角色 / 场景 / 历史 / 手动
├── 候选图网格
└── 底部操作:取消 / 添加已选
```

候选图卡片显示:

- 图片
- 名称
- 类型
- 推荐原因,例如“镜头文案提到秦昭”“当前镜头已绑定该场景”
- 是否已添加

手动图入口:

- 在“手动”tab 中提供“登记项目资产图”入口。
- 用户只能登记当前项目资产域内的 URL 或对象 key;任意外链会被后端拒绝。
- 登记成功后,新图立即出现在候选列表并默认选中。
- 登记失败时展示错误文案,不改变当前已选参考图。

### 5.3 `@` 引用下拉

在提示词 textarea 中输入 `@` 后,系统弹出下拉选项。选项来自当前镜头“已选参考图”,而不是全部候选图。

选择项显示:

```text
缩略图  @秦昭        角色
缩略图  @长安殿      场景
缩略图  @上一镜尾帧  历史
```

交互规则:

- 输入 `@` 后展示全部已选参考图。
- 输入 `@秦` 后按名称过滤。
- 键盘上下选择,Enter 确认,Esc 关闭。
- 选择后插入可读文本,例如 `@秦昭`。
- 若名称重复,前端显示和插入带后缀的别名,例如 `@秦昭-角色`、`@秦昭-上传图`。
- 如果用户删除某张参考图,且提示词中仍有对应引用,系统提示“提示词中仍引用该图片,是否同时移除引用?”。

本期 textarea 内不做不可编辑 token。引用关系在提交时由前端扫描和结构化 payload 保存。

## 6. 绑定规则

### 6.1 参考图唯一标识

每张参考图需要稳定唯一的 `id`:

```text
scene:{scene_id}
character:{character_id}
history:{render_id}
manual:{asset_id}
```

用户看到的是 `alias`,例如“秦昭”“长安殿”“上一镜尾帧”。系统保存的是 `mention_key`,即稳定 `id`。

### 6.2 提交快照

确认生成视频时保存:

- `prompt`:用户编辑后的完整提示词。
- `references`:按当前排序后的参考图列表。
- `reference_mentions`:提示词中实际出现的引用。
- `reference_binding_text`:后端生成给模型看的绑定说明。

示例:

```json
{
  "prompt": "远景，@长安殿 内，@秦昭 从门外走入画面。",
  "references": [
    {
      "id": "scene:scene01",
      "kind": "scene",
      "source_id": "scene01",
      "name": "长安殿",
      "alias": "长安殿",
      "image_url": "https://example.com/scene.png"
    },
    {
      "id": "character:char01",
      "kind": "character",
      "source_id": "char01",
      "name": "秦昭",
      "alias": "秦昭",
      "image_url": "https://example.com/char.png"
    }
  ],
  "reference_mentions": [
    {
      "mention_key": "scene:scene01",
      "label": "长安殿"
    },
    {
      "mention_key": "character:char01",
      "label": "秦昭"
    }
  ]
}
```

后端任务组装 provider prompt 时追加:

```text
参考图绑定:
图片1 = @长安殿,作为场景空间、光线与建筑风格参考。
图片2 = @秦昭,作为人物外观、服装与表情参考。
```

如果提示词中没有显式 `@` 引用,仍保留参考图,按当前顺序传给模型,并生成普通说明:

```text
参考图:
图片1 = 长安殿 场景参考。
图片2 = 秦昭 角色参考。
```

## 7. 自适应进度条设计

### 7.1 产品表现

生成中展示:

```text
镜头生成中                           62%
预计剩余约 35 秒 · 基于最近 20 条同类任务
[ progress bar ]
```

当历史样本不足:

```text
镜头生成中                           38%
预计剩余约 1 分 20 秒 · 使用默认估算
```

失败时:

```text
生成失败                             62%
输入图疑似包含敏感内容,请更换参考图后重试
```

### 7.2 估算原则

- `jobs.progress` 保持真实任务进度,仍由后端任务通过 `update_job_progress` 写入。
- 新增 `display_progress` 作为前端展示进度。
- 非终态最高显示到 95%,成功才显示 100%。
- 失败时停止在最后一次展示进度,并展示错误。
- 估算只影响 UI,不影响任务状态、重试、锁定和导出逻辑。

### 7.3 历史样本

默认取最近 20 条成功任务,可配置:

```text
JOB_PROGRESS_HISTORY_SIZE=20
JOB_PROGRESS_ESTIMATE_CAP=95
JOB_PROGRESS_ESTIMATE_MIN_SECONDS=10
```

同类任务分组:

| 任务类型 | 分组维度 |
| --- | --- |
| `gen_shot_draft` | kind |
| `render_shot` | kind + model |
| `render_shot_video` | kind + model_type + resolution + duration |
| 角色/场景生成 | kind |

推荐使用 P75 耗时作为估算总时长,比平均值更能容忍慢任务,避免进度条过早冲到 95%。

### 7.4 展示进度计算

```text
elapsed_seconds = now - job.created_at
estimated_total_seconds = P75(recent_success_durations)
estimated_progress = elapsed_seconds / estimated_total_seconds * 100
display_progress = max(job.progress, estimated_progress)
display_progress = min(display_progress, 95)  # 非终态
```

终态规则:

- `succeeded`: `display_progress = 100`
- `failed`:保持最后估算值或真实 `progress`
- `canceled`:保持取消前进度

当历史样本不足时,使用配置默认值:

```text
render_shot_video.fast.480p.default_seconds=90
render_shot_video.standard.720p.default_seconds=180
gen_shot_draft.default_seconds=45
```

## 8. 状态与错误处理

### 8.1 参考图错误

| 场景 | 反馈 |
| --- | --- |
| 未选择任何参考图就生成 | 禁用按钮并提示“至少保留 1 张参考图后才能生成视频” |
| 超出参考图上限 | 禁用继续添加,提示上限 |
| 手动图登记失败 | toast 展示失败原因,当前草稿不变 |
| 手动图不是项目资产 URL | 表单内提示“请先将图片放入当前项目资产库” |
| 参考图资源已失效 | 卡片置灰,提交时阻止并提示重新选择 |
| 删除被提示词引用的图 | 二次确认是否同步移除引用 |

### 8.2 `@` 引用错误

| 场景 | 反馈 |
| --- | --- |
| 输入 `@` 但无已选参考图 | 下拉显示“先添加参考图” |
| 引用文字被用户手动改坏 | 提交时只保存能匹配的引用,不阻止生成 |
| 同名引用 | 自动使用去重别名 |

### 8.3 进度估算错误

| 场景 | 反馈 |
| --- | --- |
| 无历史样本 | 使用默认估算,文案标注“使用默认估算” |
| job 缺少 created_at | 退回真实 `jobs.progress` |
| provider 长时间无回调 | display progress 到 95 后停住,文案展示“等待服务返回结果” |

## 9. 配置项

建议新增配置:

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| `SHOT_REFERENCE_MAX_COUNT` | `6` | 单个镜头最多参考图数量 |
| `JOB_PROGRESS_HISTORY_SIZE` | `20` | 估算进度使用最近 N 条成功任务 |
| `JOB_PROGRESS_ESTIMATE_CAP` | `95` | 非终态展示进度上限 |
| `JOB_PROGRESS_ESTIMATE_MIN_SECONDS` | `10` | 估算总时长下限 |

前端可通过后端 settings 或构建时 env 获取 `SHOT_REFERENCE_MAX_COUNT`。进度估算应以后端返回为准,前端不自行读取历史 job。

## 10. 数据与接口建议

### 10.1 新增/扩展接口

```text
GET  /api/v1/projects/{project_id}/shots/{shot_id}/reference-candidates
POST /api/v1/projects/{project_id}/shots/{shot_id}/reference-assets
GET  /api/v1/jobs/{job_id}
```

`GET /reference-candidates` 返回当前镜头可选参考图。

`POST /reference-assets` 负责登记手动项目资产图。后端必须校验 URL / object key 属于当前项目资产域,登记成功后返回标准参考图对象,由前端加入当前草稿。

`GET /jobs/{job_id}` 扩展展示字段:

```json
{
  "progress": 35,
  "display_progress": 62,
  "elapsed_seconds": 48,
  "estimated_total_seconds": 78,
  "estimated_remaining_seconds": 30,
  "estimated_source": "recent_20"
}
```

### 10.2 扩展提交结构

镜头视频生成提交结构扩展:

```json
{
  "prompt": "远景，@长安殿 内，@秦昭 从门外走入画面。",
  "references": [],
  "reference_mentions": [],
  "duration": 5,
  "resolution": "480p",
  "model_type": "fast"
}
```

兼容要求:

- `reference_mentions` 可选。
- 老前端不传 `reference_mentions` 时,后端按普通参考图处理。
- 后端保存历史版本时必须保留原始 `prompt`、`references` 和 `reference_mentions`。

## 11. 验收标准

### 11.1 参考图与提示词

- 在镜头生成页,参考图缩略图显示在提示词上方。
- 用户可以从候选图库追加参考图。
- 用户可以登记一张当前项目资产图并加入当前镜头草稿。
- 用户可以删除、排序参考图。
- 输入 `@` 后出现仅包含当前已选参考图的下拉。
- 选择下拉项后,提示词中插入对应引用。
- 提交生成后,历史版本中能看到当时的提示词、参考图列表和引用关系。
- 删除被引用参考图时有明确确认反馈。

### 11.2 进度条

- 生成任务进行中时展示 `display_progress`。
- 有最近 N 条同类成功任务时,展示“基于最近 N 条同类任务”。
- 样本不足时使用默认估算,并展示“使用默认估算”。
- 非终态进度不超过配置上限,默认 95%。
- 成功后进度为 100%,失败后展示错误原因。
- 刷新页面后重新接管 in-flight job,进度估算继续生效。

### 11.3 兼容性

- 不影响现有草稿生成、视频生成、历史版本切换、锁定最终版。
- 不改变 `project.stage` 写入路径。
- 不阻塞 HTTP 请求等待远程 AI 完成。
- 老版本历史记录没有 `reference_mentions` 时仍可正常展示。

## 12. 后续可扩展方向

- 将 textarea 升级为轻量富文本 token 编辑器,让 `@引用` 变成不可编辑 chip。
- 支持参考图分组,例如“人物参考”“场景参考”“风格参考”。
- 支持同一参考图指定作用域,例如只影响人物、只影响背景、只影响构图。
- 支持批量镜头复用上一镜头参考图。
- 根据任务参数进一步细分进度样本,例如模型版本、图片数量、视频秒数。
