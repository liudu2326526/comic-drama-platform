# 漫剧生成平台产品设计文档

## 1. 产品概述

### 1.1 产品定位

漫剧生成平台是一套面向内容创作者、小说改编团队、短剧工作室的 AI 生产工具，目标是把一段小说内容快速转化为可导出的漫剧短视频。

平台聚焦一条标准化生产链路：

`小说内容输入 → 自动生成分镜草案 → 在分镜设计中确定镜头数 → 固定角色参考资产 → 固定场景参考资产 → 镜头生成 → 导出短视频`

### 1.2 核心价值

- 降低小说改编为漫剧视频的制作门槛
- 把原本依赖人工编剧、分镜、美术、剪辑的流程标准化
- 提升批量内容生产效率，适合做章节连载和日更

### 1.3 目标用户

- 网文改编团队
- 漫剧内容工作室
- 短视频剧情号运营者
- 小说 IP 孵化团队

## 2. 产品目标

### 2.1 业务目标

- 让用户在 10 到 20 分钟内完成一次漫剧短视频生产
- 支持单项目按剧情复杂度产出合适数量的镜头内容
- 降低角色不一致、背景混乱、镜头风格不统一的问题

### 2.2 用户目标

- 输入原始小说文本后，快速获得结构化分镜
- 锁定主角设定，保证后续镜头人设统一
- 通过场景模板和场景参考资产降低生成成本与不稳定性
- 在关键节点可人工微调，而不是完全黑盒生成

## 3. 范围与前提

### 3.1 本期产品范围

- 单次输入一段小说内容
- 自动生成分镜草案，并在分镜设计阶段确定镜头数
- 支持 1 到 3 个主要角色，其中主角形象固定
- 场景采用模板化与参考资产结合的方案，不做完全自由生成
- 生成镜头画面并串联导出短视频

### 3.2 默认前提

- 视频形态优先为竖屏短视频，建议默认 `9:16`
- 每个镜头默认生成 1 张主画面，可扩展为镜头动画
- 输出为“图像镜头拼接视频”模式，后续可叠加字幕、配音、转场
- 主角角色和核心场景在确认后会沉淀为可复用参考资产，后续生成默认强约束复用

### 3.3 非本期重点

- 复杂角色关系网编辑
- 自由级时间轴剪辑
- 高度电影化运镜编辑
- 多剧集协同管理

## 4. 核心业务流程

### 4.1 端到端流程

1. 用户创建项目并输入小说内容
2. 系统解析内容并抽取人物、场景、情节节点
3. 系统自动生成分镜草案并给出建议镜头数
4. 用户确认主角角色设定，生成角色参考资产并锁定角色形象
5. 系统为每个镜头匹配场景模板并生成场景参考资产
6. 系统逐镜头生成画面
7. 用户检查镜头结果并对异常镜头重试或微调
8. 系统按镜头顺序合成为短视频
9. 用户导出视频文件

### 4.2 分阶段业务流

### 阶段 A：小说输入

输入：

- 小说正文
- 可选标题
- 可选时代/风格标签

系统处理：

- 文本清洗
- 段落切分
- 角色名识别
- 场景与情绪抽取
- 剧情主线提炼

输出：

- 项目基础信息
- 原始内容结构化结果
- 可用于分镜生成的剧情摘要

### 阶段 B：分镜设计与镜头数确定

输入：

- 结构化剧情内容

系统处理：

- 按起承转合切分情节
- 生成初始分镜卡片与建议镜头数
- 为每个镜头补充镜头描述、人物出场、场景标签、情绪标签
- 支持用户合并、拆分、删除、新增镜头

输出：

- 已确认镜头数的分镜列表

业务规则：

- 系统先给出建议镜头数，不强制固定数量
- 用户在分镜设计阶段确认最终镜头数
- 若原文过短，可通过补足过渡镜头增强节奏
- 若原文信息过密，优先抽取核心冲突与关键事件

### 阶段 C：主角角色固定

输入：

- 系统识别出的角色列表
- 用户选择的主角

系统处理：

- 生成主角角色描述 string
- 生成角色参考图
- 生成角色视频形象参考信息
- 汇总性别、年龄感、发型、服饰、体型、气质、画风等字段
- 锁定主角视觉设定，后续所有镜头复用同一角色锚点

输出：

- 主角角色描述 string
- 角色参考图片资产
- 角色视频形象参考信息
- 角色一致性生成约束

业务规则：

- 主角设定确认后，镜头生成阶段默认不可随意漂移
- 允许用户在生成前重新编辑一次角色描述与参考资产
- 配角可以弱约束，主角必须强约束
- 角色描述 string 是后续生成图片和视频时的基础提示信息
- 角色图片和视频参考信息用于后续视频生成阶段的人设对齐

### 阶段 D：场景固定与模板化

输入：

- 每个镜头的场景标签
- 预置场景模板库

系统处理：

- 按室内/室外、古代/现代、白天/夜晚等维度匹配模板
- 允许同一场景在多个镜头中复用
- 支持用户手动替换场景模板
- 生成场景描述 string
- 生成场景参考图
- 生成场景视频形象参考信息

输出：

- 每个镜头关联的场景模板
- 场景描述 string
- 场景参考图片资产
- 场景视频形象参考信息

业务规则：

- 场景优先模板匹配而非自由生成
- 保证同场景镜头的空间连续性
- 场景模板需支持风格统一
- 场景描述 string 是后续镜头生成和视频生成的基础环境提示
- 场景图片和视频参考信息用于保证场景连续性与空间一致性

### 阶段 E：镜头生成

输入：

- 分镜描述
- 角色描述 string 与角色参考资产
- 场景描述 string 与场景参考资产
- 风格参数

系统处理：

- 生成镜头画面
- 应用角色一致性约束
- 应用场景一致性约束
- 记录生成版本与失败状态

输出：

- 全部镜头的最终候选画面

业务规则：

- 支持单镜头重生成
- 支持局部修改文案后重生成
- 支持批量生成和断点续跑

### 阶段 F：导出短视频

输入：

- 已确认的镜头列表
- 镜头顺序
- 时长与转场参数

系统处理：

- 依序拼接镜头
- 生成默认转场与镜头停留时长
- 导出 MP4 短视频

输出：

- 可下载视频文件
- 导出记录

业务规则：

- 未完成镜头不得进入最终导出
- 导出前需提示缺失镜头或失败镜头
- 支持导出标清/高清两个等级

## 5. 核心页面架构

建议采用“项目工作台”型产品结构，核心页面如下：

1. 项目列表页
2. 新建项目页
3. 分镜工作台
4. 角色设定页
5. 场景设定页
6. 镜头生成页
7. 导出页

## 6. 页面信息设计

### 6.1 项目列表页

页面目标：

- 管理所有漫剧项目
- 快速进入未完成项目继续生产

页面模块：

- 顶部搜索与筛选
- 项目卡片/表格列表
- 新建项目按钮
- 项目状态标签

核心字段：

- 项目名称
- 原始小说标题
- 当前阶段
- 镜头完成数
- 最近更新时间
- 导出状态

关键操作：

- 新建项目
- 继续编辑
- 复制项目
- 删除项目

### 6.2 新建项目页

页面目标：

- 完成小说内容输入与项目初始化

页面模块：

- 项目基础信息
- 小说文本输入框
- 风格选择
- 生成预检查区

核心字段：

- 项目名称
- 小说正文
- 漫剧风格
- 时代背景
- 输出比例

关键操作：

- 保存草稿
- 开始拆分分镜

交互重点：

- 支持大段文本输入
- 实时显示字数
- 文本过短或过长时给出提示

### 6.3 分镜工作台

页面目标：

- 承接自动分镜结果，让用户确认镜头结构与最终镜头数

页面模块：

- 顶部流程进度条
- 左侧镜头列表
- 中间镜头详情编辑区
- 右侧剧情摘要与角色提示

镜头卡字段：

- 镜头编号
- 镜头标题
- 剧情描述
- 出场角色
- 场景标签
- 情绪标签
- 生成状态

关键操作：

- 调整镜头文案
- 标记重点镜头
- 单镜头重拆
- 确认进入角色设定

交互重点：

- 所有镜头必须可快速纵览
- 当前镜头与上下文镜头要联动展示
- 需要清晰标识“未确认修改”

### 6.4 角色设定页

页面目标：

- 锁定主角形象并建立角色一致性基础

页面模块：

- 角色列表
- 主角参考资产卡
- 角色预览图
- 角色视频形象参考信息
- 设定字段表单

角色参考资产字段建议：

- 角色名
- 主/配角标记
- 角色描述 string
- 性别
- 年龄感
- 发色发型
- 服装风格
- 身材特征
- 性格气质
- 画风关键词
- 禁止漂移描述
- 角色参考图
- 角色视频形象参考信息

关键操作：

- 指定主角
- 编辑角色参考资产
- 保存角色设定
- 重新生成角色预览
- 确认角色参考资产

交互重点：

- 主角必须突出显示并锁定
- 修改主角设定时，要提示会影响后续镜头一致性
- 角色预览、角色描述 string、视频形象参考需要双向校验

### 6.5 场景设定页

页面目标：

- 为镜头批量或逐个匹配场景模板并沉淀场景参考资产

页面模块：

- 场景模板库
- 场景筛选器
- 当前镜头预览
- 已匹配结果列表
- 场景参考资产预览

模板字段：

- 模板名称
- 场景类型
- 时代标签
- 时间标签
- 风格标签
- 可适配镜头规模
- 场景描述 string
- 场景参考图
- 场景视频形象参考信息

关键操作：

- 自动匹配模板
- 手动替换模板
- 批量应用到相似镜头
- 确认场景参考资产

交互重点：

- 相同场景的镜头要支持一键复用场景参考资产
- 替换模板后要提示影响的镜头范围
- 模板预览和场景描述 string 要足够直观，避免用户靠名字猜

### 6.6 镜头生成页

页面目标：

- 批量完成镜头画面生成，并对失败镜头进行修复

页面模块：

- 顶部批量生成控制条
- 镜头生成队列
- 选中镜头详情
- 画面预览区
- 失败重试区

每个镜头展示信息：

- 镜头编号
- 当前提示词摘要
- 角色约束状态
- 场景约束状态
- 生成进度
- 最新结果图

关键操作：

- 批量生成全部镜头
- 单镜头重新生成
- 局部修改后重生成
- 锁定某张结果为最终版

交互重点：

- 生成过程必须可见、可中断、可恢复
- 对失败原因做明确分类，如角色冲突、背景缺失、生成超时
- 重生成前要保留旧版本，避免用户丢失可用结果

### 6.7 导出页

页面目标：

- 完成镜头检查、视频参数设置和最终导出

页面模块：

- 镜头完成检查区
- 视频参数设置区
- 导出预览区
- 导出任务列表

核心字段：

- 分辨率
- 比例
- 单镜头时长
- 转场样式
- 文件名

关键操作：

- 检查缺失镜头
- 预览合成顺序
- 发起导出
- 下载成片

交互重点：

- 导出前必须做完整性校验
- 导出任务需要异步处理并展示进度
- 导出成功后要保留历史版本和下载记录

## 7. 关键交互设计

### 7.1 流程化导航

- 顶部使用步骤条展示当前处于“输入内容 / 拆分分镜 / 角色设定 / 背景匹配 / 镜头生成 / 导出”
- 已完成步骤可回退，未完成步骤不可跳过进入下一步

### 7.2 人工可干预点

平台不能是完全黑盒，至少保留以下人工干预点：

- 分镜文案可修改
- 主角角色描述与参考资产可确认
- 场景模板与参考资产可替换
- 失败镜头可单独重试
- 导出前镜头顺序可检查

### 7.3 状态反馈

- 所有 AI 任务都要有明确状态：未开始、处理中、成功、失败
- 长任务必须显示进度、预计耗时、失败原因
- 批量任务中要支持查看单条结果

### 7.4 一致性保障

- 主角角色锁定后，所有镜头都显示“角色约束已启用”
- 相同场景使用相同场景参考资产时，显示“场景复用中”
- 若用户修改主角设定，系统提示哪些镜头建议重新生成
- 若用户修改场景参考资产，系统提示哪些镜头建议重新生成

### 7.5 容错与恢复

- 任一步骤都支持保存草稿
- 生成中断后可从上次状态继续
- 重生成不覆盖历史版本，支持回退

## 9. MVP 建议

首期建议把功能压缩到最小闭环，先跑通生产链路：

- 输入小说文本并创建项目
- 自动生成分镜草案并确认镜头数
- 锁定 1 个主角角色参考资产
- 自动匹配场景模板并生成场景参考资产
- 生成镜头静帧
- 拼接导出短视频

首期可以暂缓的能力：

- 多主角复杂关系
- 配音与字幕自动生成
- 动态运镜与口型动画
- 多版本批量导出

## 10. 交付建议

为了方便后续进入原型和研发阶段，建议下一步继续拆成 3 份文档：

1. 页面原型说明文档
2. 后端领域模型与状态机文档
3. AI 生成链路与提示词策略文档

## 11. 一句话结论

这个平台的产品核心不是“让 AI 自由作画”，而是把小说改编流程标准化为一条可控的生产流水线：先把剧情拆对，再沉淀稳定的角色参考资产和场景参考资产，最后再去做镜头生成与视频导出，这样才能真正适合规模化生产漫剧内容。
